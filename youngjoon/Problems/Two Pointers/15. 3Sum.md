# 15. 3Sum


Given an integer array nums, return all the triplets [nums[i], nums[j], nums[k]] such that i != j, i != k, and j != k, and nums[i] + nums[j] + nums[k] == 0.

Notice that the solution set must not contain duplicate triplets.

 

> **Example 1**
> 
> Input: nums = [-1,0,1,2,-1,-4]
> 
> Output: [[-1,-1,2],[-1,0,1]]
> 
> Explanation: 
> 
> nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0.
> 
> nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0.
> 
> nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0.
> 
> The distinct triplets are [-1,0,1] and [-1,-1,2].


Notice that the order of the output and the order of the triplets does not matter.

> **Example 2**
> Input: nums = [0,1,1]
> 
> Output: []
> 
> Explanation: The only possible triplet does not sum up to 0.

> **Example 3**
> 
> Input: nums = [0,0,0]
> 
> Output: [[0,0,0]]
> 
> Explanation: The only possible triplet sums up to 0.

---

1. 어떤 식으로 3개의 숫자 조합을 이용할까?
1) 조합을 이용해서 접근 (DFS : Backtracking) -> O(N^3)
2) 하나의 값을 고정한 Two Sum 방식 접근

a + b + c = 0 -> b + c  = - a 

2. Two Sum 이라면, 어떤 풀이로 ?
1) Hashmap -> O(N)이지만, 중복 제거에서 불편
    - 외부 Loop O(N) + 내부 Two sum O(N) -> O(N^2)  
    - Set 에 넣고 O(log K) + vector 비교
    - 총 Time complexity : O(N^2 log N)
2) Two pointer -> O(N)이지만, 정렬이 필요함
    - 정렬 O(N log N) + 탐색 O(N^2) + 중복제거 O(1)
    - 총 Time complexity : O(N^2)

그러므로 2) 방식을 이용.

참고로 이 문제는 index 를 return 해야하는 문제가 아니므로, 정렬해도 상관없음!

3. Two pointer 방식에서 주의할 점은?

이 문제의 핵심은 아래 코드.

```cpp
int sum = nums[left] + nums[i] + nums[right] 
```
여기서 for loop으로 하나를 고정하고, 그 안에서 while 로 순회.

---

### Solution

```cpp
class Solution {
public:
    vector<vector<int>> threeSum(vector<int>& nums) {
        sort(nums.begin(), nums.end());
        vector<vector<int>> results;

        int n = nums.size();

        for (int i = 0; i < n; i++) {
            // i 중복 제거
            if (i > 0 && nums[i] == nums[i - 1]) {
                continue;
            }

            int left = i + 1;
            int right = n - 1;

            while (left < right) {
                int sum = nums[i] + nums[left] + nums[right];

                if (sum == 0) {
                    results.push_back({nums[i], nums[left], nums[right]});

                    left++;
                    right--;

                    // left 중복 제거
                    while (left < right && nums[left] == nums[left - 1]) {
                        left++;
                    }

                    // right 중복 제거
                    while (left < right && nums[right] == nums[right + 1]) {
                        right--;
                    }
                }
                else if (sum < 0) {
                    left++;
                }
                else {
                    right--;
                }
            }
        }
        return results;
    }
};
```

4. 핵심 코드

```cpp
int left = i+1;
```

만약 i 를 left 와 right 의 중간으로 잡게되면,(left = 0; 으로 진행 시)
중복으로 순회하는 부분이 생기지 않게 조건을 달아줘야한다.

깔끔하게 풀기위해서는 위와 같이 i+1 로 접근하는 것이 좋음.

5. 중복 제거 방식은?
1) set 이용하기 -> 매번 set.insert()호출 비효율적
2) index를 이용하기 
-> sum == 0 조건을 만족한 후에, index 이동. 
이 때, 중복된 nums[index]가 있다면 비효율적이므로, (+중복입력 오답)
다른 값이 나올 때 까지 index 이동이 필요하다.
