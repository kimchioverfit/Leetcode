# 238. Product of Array Except Self

Given an integer array nums, return an array answer such that answer[i] is equal to the product of all the elements of nums except nums[i].

The product of any prefix or suffix of nums is guaranteed to fit in a 32-bit integer.

You must write an algorithm that runs in O(n) time and without using the division operation.


> **Example 1**
> 
> Input: nums = [1,2,3,4]
> 
> Output: [24,12,8,6]


> **Example 2**
> 
> Input: nums = [-1,1,0,-3,3]
> 
> Output: [0,0,9,0,0]

### Solution
```cpp
class Solution {
public:

    vector<int> productExceptSelf(vector<int>& nums) {
        int tm = 1;
        vector<int> results;
        unordered_map<int,int> um;
        int cnt = 0;
        for(int i=0; i<nums.size(); i++){
            if(nums[i]==0){
                um[nums[i]]++;
                continue;
            }
            tm *= nums[i];
        }
        if(um[0]>1){
            results.assign(nums.size(),0);
            return results;
        }
        for(int i=0; i<nums.size(); i++){
            if(um[0]==1 && nums[i]!=0){
                results.push_back(0);
            }
            else if(um[0]==1 && nums[i]==0){
                results.push_back(tm);
            }
            else results.push_back(tm/nums[i]);
        }
        return results;
    }
};
```

Prefixsum 에서 대표격인 문제 중 하나. 

우선 기본적인 방식은 전체 곱에서 자기 자신만을 제외하면 되는데, 

이 떄의 시간 복잡도를 최소화 하는 것이 관건. 

접근 사고의 흐름은 아래와 같음. 

1. 전체 곱에서 자기 자신만 제외하는 방법은?
1) 다 곱해준 다음, 자기 자신으로 나누기 -> 0의 갯수에 따라 예외처리 해주는 방식으로 접근 시 O(1)
2) 자기 자신을 기준으로, 연산을 분리해주기 

위 2가지 방식 모두 O(1) 시간내로 가능하지만, 1의 방식은 문제의 출제 의도에 벗어나므로 2의 풀이로 접근한다.
 
2. 연산을 어떻게 분리할까? 

여기서는 좌/우로 분리해서 접근한다. (순차적이므로)

<u>nums[0] * nums[1] * ... </u> * Target *<u> ... * nums[n-1]</u>
Left * Target * Right

Left : [nums[0], nums[0] * nums[1], nums[0] * nums[1] * nums[2], ... , <u>... * nums[n-2]</u>] 
(자기 자신은 제외해야하므로 n-2까지)

Right : [nums[1] * nums[2] * nums[n-1], ..., nums[0]]


### Solution

```cpp
class Solution {
public:
    vector<int> productExceptSelf(vector<int>& nums) {
        vector<int> left(nums.size(),1);
        vector<int> right(nums.size(),1);
        for(int i=0; i<nums.size()-1; i++){
            left[i+1] = left[i]*nums[i];
        }
        for(int i=nums.size()-1; i>=1; i--){
            right[i-1] = right[i]*nums[i];
        }
        vector<int> results(nums.size(),0);
        for(int i=0; i<nums.size(); i++){
            results[i] = left[i]*right[i];
        }
        return results;
    }
};
```

3. 여기서 더 개선할 점은 없을까? 
-> 공간 복잡도 측면에서 개선할 수 있다. 어차피 results는 left * right 기 때문에, 
곱셈을 두 번 나눠서 좌측 누적곱, 우측 누적곱을 계산하면 별도 곱셈용 변수만 있으면 딘다.

### Solution(Advanced)
```cpp
class Solution {
public:
    vector<int> productExceptSelf(vector<int>& nums) {
        int n = nums.size();
        vector<int> result(n, 1);

        int left = 1;
        for(int i = 0; i < n; ++i) {
            result[i] = left;
            left *= nums[i];
        }

        int right = 1;
        for(int i = n - 1; i >= 0; --i) {
            result[i] *= right;
            right *= nums[i];
        }

        return result;
    }
};
```
